### GlowUS 에이전트 vs OpenClaw 에이전트

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   GlowUS 에이전트 = 정규직 (조직 구성원)                         │

│   • 페르소나 (정체성)                                           │
│   • 소속팀 + 상사                                               │
│   • 5-Layer Memory (기억)                                       │
│   • 레벨 + 경험치 (성장)                                        │
│   • Governance (통제/책임)                                      │
│                                                                 │
│   OpenClaw = 외주 도구함 (플러그인)                              │
│   ────────────────────────────────                              │
│   • 기억 없음                                                   │
│   • 소속 없음                                                   │
│   • 성장 없음                                                   │

```

**비유:**

| 비유 | GlowUS 에이전트 | OpenClaw |
|------|----------------|----------|
| **소프트웨어** | 포토샵 본체 | 플러그인/필터 모음 |
| **IDE** | VSCode | Extension 마켓 |
| **스마트폰** | iOS | App Store 앱들 |
| **회사** | 정규직 직원 | 외부 SaaS 구독 |

### 실제 동작 흐름

```
GlowUS 리서처 에이전트 (정규직)
│
├── 내 스킬 목록:
│   │
│   ├── 🏠 GlowUS Native (자체 앱)
│   │   ├── ai_sheet
│   │   ├── ai_docs
│   │   └── image_gen
│   │
│   └── 🦞 OpenClaw Plugin (외주 도구)
│       ├── web_search
│       ├── scrape_website
│       └── browser_automation
│
├── 태스크: "경쟁사 분석해줘"
│
└── 실행:
    1. 내가 판단 → "웹 검색 필요하네"
    2. OpenClaw web_search 플러그인 호출
    3. 결과 받아서 → 내 Memory에 저장
    4. ai_sheet로 데이터 정리
    5. ai_docs로 리포트 작성
    6. 완료 → 경험치 획득
```

**핵심:**
- **판단/기억/책임** → GlowUS 에이전트
- **실행** → OpenClaw 플러그인 (필요할 때만)

```
┌─────────────────────────────────────────────────────────────┐
│                   Agent Memory Architecture                  │

│                                                             │
│  Layer 1: Private Memory (개인 대화)                         │
│  ─────────────────────────────────────                      │
│  "사용자가 간결한 보고서를 선호함"                             │
│  "월요일 아침은 피드백이 까다로움"                             │
│                                                             │
│  Layer 2: Meeting Memory (회의 기록)                         │
│  ─────────────────────────────────────                      │
│  "Q1 전략 회의에서 결정된 방향"                               │
│  "클라이언트 요구사항 변경 내역"                              │
│                                                             │
│  Layer 3: Team Memory (팀 지식)                              │
│  ─────────────────────────────────────                      │
│  "우리 팀 브랜드 가이드라인"                                  │
│  "프로젝트 히스토리 & 의사결정 로그"                          │
│                                                             │
│  Layer 4: Injected Memory (주입 지식)                        │
│  ─────────────────────────────────────                      │
│  "회사 정책 문서"                                            │
│  "제품 매뉴얼 & 스펙"                                        │
│                                                             │
│  Layer 5: Execution Memory (실행 기록)                       │
│  ─────────────────────────────────────                      │
│  "지난 번 리포트 피드백 반영 내역"                            │
│  "실패한 접근 방식 기록"                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘

OpenClaw: 매번 처음부터 설명해야 함
GlowUS: 다 기억하고, 학습하고, 성장함
```

### 핵심 API 인터페이스

```typescript
// types/openclaw.ts

// OpenClaw Gateway 이벤트
interface OpenClawEvent {
  type: 'message' | 'tool_start' | 'tool_end' | 'error';
  sessionId: string;
  timestamp: Date;
  data: any;
}

// GlowUS → OpenClaw 요청
interface OpenClawRequest {
  action: 'send_message' | 'invoke_skill' | 'list_skills';
  sessionId?: string;
  skill?: string;
  params?: Record<string, any>;
  context?: GlowUSContext;  // Memory, Governance 컨텍스트
}

// GlowUS 컨텍스트 (OpenClaw에 주입)
interface GlowUSContext {
  userId: string;
  agentId: string;
  teamId?: string;

  // 5-Layer Memory에서 추출된 관련 컨텍스트
  memoryContext: string;

  // Governance 정책
  policies: {
    maxCost: number;
    allowedSkills: string[];
    requireApproval: string[];
  };
}

// OpenClaw 스킬 → GlowUS 스킬 변환 결과
interface AdaptedSkill {
  id: string;                    // openclaw_${name}
  name: string;
  description: string;
  source: 'openclaw';
  originalSkill: OpenClawSkill;
```
